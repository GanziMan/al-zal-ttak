# 알공딱 - AI 공시 분석

## 모델

- **Google ADK** (Agent Development Kit)
- **모델:** `gemini-2.5-flash`
- **파일:** `backend/app/agents/categorizer.py`

## 분석 결과 구조

```json
{
  "category": "호재",
  "importance_score": 75,
  "summary": "삼성전자가 자기주식 1000억원 규모 취득을 결정했습니다.",
  "action_item": "주가 지지 효과 기대, 매수 검토"
}
```

## 분류 기준

### 호재 (주가 상승 요인)
- 대규모 수주/계약 체결, 매출/영업이익 증가, 흑자 전환
- 자사주 매입/소각, 배당 확대, 무상증자
- 신사업 진출, 기술 특허, M&A (인수하는 쪽)
- CB/BW 조기상환, 부채비율 개선

### 악재 (주가 하락 요인)
- 유상증자(희석), CB 발행, 영업손실/적자 전환
- 소송 패소, 과징금, 횡령/배임
- 감사의견 거절/한정, 상장폐지 사유
- 대규모 투자손실, 최대주주 지분 매각

### 중립
- 임원 변동(대표 제외), 정기 주총, 사외이사 선임
- 소규모 지분 취득/처분, 소규모 시설투자

### 단순정보
- 사업보고서, 반기/분기보고서 제출
- 공정공시, IR자료
- 임원/주요주주 소유상황보고

## 중요도 점수 기준

| 점수 | 설명 |
|------|------|
| 80~100 | 즉각적 주가 영향 (M&A, 상장폐지, 분식회계 등) |
| 60~79 | 단기~중기 영향 (실적 발표, 대규모 수주 등) |
| 40~59 | 참고 정보 (임원 변동, 배당 결정 등) |
| 20~39 | 관심 수준 (소규모 거래, 주총 결과 등) |
| 0~19 | 형식적 공시 (정기보고서 제출 등) |

## 분석 흐름

1. 유저가 공시 목록 조회
2. 각 공시의 `rcept_no`로 DB 캐시 확인
3. 캐시 미스 → 백그라운드 태스크로 AI 분석 실행
4. Gemini에 종목명 + 공시 제목 + 내용 전달
5. 구조화된 분석 결과 반환 → DB에 캐시 저장
6. 프론트에서 5초 간격 폴링으로 분석 완료 감지

## 분석 입력

```
[종목명] 삼성전자
[공시 제목] 주요사항보고서(자기주식취득결정)
[공시 내용]
삼성전자 주요사항보고서(자기주식취득결정) (접수일: 20260322)
```

> 현재 DART API에서 공시 본문을 가져오지 않고 제목 기반으로 분석합니다.
> 향후 공시 본문 크롤링을 추가하면 분석 품질이 향상됩니다.
